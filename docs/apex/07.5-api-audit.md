## Assessment

This is a CLI tool, not an API service. The "API" surface is:

1. **speedtest.net XML API** (consumer) — fetches servers, sends pings, downloads/uploads
2. **CLI output formats** (producer) — JSON, CSV, human-readable
3. **Shell completions** (producer) — generated by `clap_complete`

### speedtest.net XML API Consumption

| Aspect | Status | Notes |
|--------|--------|-------|
| Server list XML | ✅ | `quick-xml` deserialization with proper struct mapping |
| Error handling | ✅ | Malformed XML returns `SpeedtestError::Context` |
| Timeout handling | ✅ | Configurable 1-300s timeout via `reqwest` |
| TLS | ✅ | `rustls-tls` (no OpenSSL) |
| User-Agent | ⚠️ | Spoofs Chrome 120 on macOS (`http.rs`) — transparency concern |

### CLI Output API (JSON/CSV Contracts)

| Aspect | Status | Notes |
|--------|--------|-------|
| JSON schema | ✅ | `TestResult` struct with `Serialize` derive |
| CSV schema | ✅ | `CsvOutput` struct, configurable delimiter |
| Breaking change protection | ⚠️ | No version field in JSON output — clients can't detect schema changes |
| Null handling | ✅ | Optional fields use `Option<T>` with `skip_serializing_if` where appropriate |

### Findings

| ID | Title | Severity | Description |
|----|-------|----------|-------------|
| API-001 | JSON output has no schema version | LOW | Consumers cannot detect breaking changes in `TestResult` structure. Fix: Add `"schema_version": "1"` field |
| API-002 | User-Agent impersonation | LOW | `http.rs` spoofs Chrome UA. Should document or use `netspeed-cli/{version}`. Fix: Consider transparent UA or document rationale |
